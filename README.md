# M-planner

기숙사 학습 플래너. 단일 HTML 파일, 서버 없이 브라우저에서 바로 동작합니다.
데이터는 이 브라우저의 localStorage에 저장됩니다.

## 지금 바로 노트북에서 쓰기

`M-planner.html` 파일을 더블클릭하면 크롬에서 열립니다. 그대로 사용하면 됩니다.
북마크해두면 편합니다.

## GitHub Pages에 올려서 URL로 쓰기

1. GitHub에서 새 저장소(repository)를 만듭니다. 예: `m-planner`
2. `M-planner.html` 파일을 올립니다. 이때 파일 이름을 **`index.html`로 바꿔서** 올리면 주소가 깔끔해집니다.
   (안 바꾸면 `.../M-planner.html`로 접근해야 함)
3. 저장소 → **Settings → Pages** 로 갑니다.
4. **Source**를 `Deploy from a branch`, **Branch**를 `main` / `/(root)`로 설정하고 Save.
5. 1~2분 뒤 `https://<your-id>.github.io/m-planner/` 주소가 생깁니다. 그 주소로 접속하면 끝.

이후 그 주소를 크롬에서 새 탭·새 창 몇 번을 열어도 데이터가 이어집니다.

## 데이터에 대해 알아둘 점

- **같은 브라우저·같은 주소**에서만 데이터가 공유됩니다. 크롬에서 저장한 건 사파리·시크릿·다른 기기에서는 안 보입니다.
- **브라우저 사이트 데이터를 지우면 플래너 데이터도 사라집니다.** 왼쪽 아래 **백업** 버튼으로 가끔 JSON 파일을 내려받아 두세요. 문제가 생기면 **복원** 버튼으로 되돌릴 수 있습니다.
- 나중에 여러 기기에서 동기화하고 싶으면 이 백업 파일이 이전(migration)의 출발점이 됩니다.

## 화면 구성

- **주간 플래너** — 활성 템플릿의 고정 일정이 배경에 깔리고, 배치된 조각을 눌러 미완→일부→완료로 순환. 하단에 요일별 달성률·순공.
- **할 일 관리** — 과목별 할 일을 세분화(토글), 각 항목에 예상 소요시간. 일부 완료 시 내일/특정날짜로 미루기.
- **통계** — 주간/월별 전환. 요일별 달성률·순공 막대(평균선), 과목별 도넛, 계획 vs 실제, 월별 잔디 히트맵.
- **순공 기록** — 오프라인 타이머로 잰 시간을 구간 입력 → 자동 계산 + 세로 타임라인.
- **월별 점검표** — 목표 진행률, 주간 회고 모아보기, 월간 회고·컨디션.
- **시간표 템플릿** — 격자 에디터로 고정 일정 관리. 학기·방학 템플릿 전환.
- **과목 관리** — 과목 추가·색 지정 (프로그래밍, 컴퓨터 시스템 일반 등 자유롭게).

## 자동 배치와 드래그

- **자동 배치** — 주간 플래너 오른쪽 위 "자동 배치" 버튼. 마감 임박·중요도 높은 할 일부터, 공부 가능 슬롯(아침자습·방과후·야자 등)에 예상 소요시간만큼 채웁니다. 슬롯보다 큰 항목은 연속된 슬롯에 걸쳐서 나눕니다. 오늘은 이미 지난 시각을 건너뛰고 남은 시간부터 채웁니다.
- **드래그** — 배치된 조각을 잡아 다른 슬롯으로 옮길 수 있습니다. 30분 단위로 스냅되고, 고정 블록이나 다른 조각과 겹치면 원위치되며 안내가 뜹니다.
- **체크** — 조각 오른쪽 위 동그라미를 누르면 미완 → 일부 → 완료로 바뀝니다. 일부 완료 시 미루기 창이 뜨고, 하단 요일별 달성률에 바로 반영됩니다.
